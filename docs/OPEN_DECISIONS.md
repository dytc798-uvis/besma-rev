# BESMA 미결정 사항 (Open Decisions)

아래 항목은 **아직 오너(또는 제품 책임자) 선택이 없다**.  
Cursor / 협업 도구는 **임의로 골라 구현하지 말고**, 필요 시 이 목록을 참고해 **사용자에게 질문**한다.

결정이 나면 **`DECISION_LOG.md`에 새 ID로 기록**하고, 본 파일에서 해당 OPEN을 “해결됨”으로 옮긴다.

---

## Cursor / 협업 도구 운영 규칙 (필수)

1. 구현 전에 **`DECISION_LOG.md` + `ARCHITECTURE_INVARIANTS.md`**를 읽는다.
2. 아래 OPEN과 맞닿은 작업이면 **사용자에게 옵션을 제시**한다.
3. 사용자가 고르면 **`DECISION_LOG.md`에 반영**하고, 여기서는 **[해결됨]** 섹션으로 옮기거나 삭제한다.
4. 새로운 갈림길이 보이면 **본 문서에 OPEN 항목을 추가**한다.
5. **Decision Trigger** — UI 구조, KPI, 정렬, 상태 표현, 사용자 흐름이 갈라지면 **임의 구현 금지**. 반드시 아래 **[DECISION REQUIRED]** 템플릿으로 질문한 뒤, 동일 내용을 본 문서에 OPEN으로 남긴다.
6. **확인 순서** — (1) `DECISION_LOG.md` (2) `ARCHITECTURE_INVARIANTS.md` (3) `OPEN_DECISIONS.md` → 없으면 질문·기록 → 답변 후 구현.

### 결정 요청 템플릿 ([DECISION REQUIRED])

대화(또는 이슈 본문)에 사용할 때는 아래 형식을 그대로 쓴다. `OPEN` 항목으로 옮길 때는 표에 **Title / Context / Options / Impact / Recommendation**을 모두 담는다.

```text
[DECISION REQUIRED]

Title:
(결정 주제)

Context:
(왜 필요한지)

Options:
A.
B.
C.

Impact:
(각 선택의 영향)

Recommendation:
(추천안)
```
