# BESMA 사용설명서

이 문서는 현재 운영 UI 기준으로 정리한 BESMA 전체 사용설명서입니다. 현장(SITE)은 실행과 업로드 중심, 본사(HQ)는 모니터링과 승인 중심으로 사용합니다.

## 시작하기

- 로그인 후 좌측 사이드바에서 필요한 메뉴를 선택합니다.
- 현장은 `대시보드`, `내 현장 문서`, `근로자의견청취`, `부적합사항`을 가장 자주 사용합니다.
- 본사는 `대시보드`, `문서취합현황`, `안전보건 방침 및 목표`, `설정`을 가장 자주 사용합니다.
- 설명서 본문은 앱의 `사용설명서` 메뉴에서도 같은 내용으로 확인할 수 있습니다.

## 오늘부터 달라진 점

- 문서취합은 `Requirement 기반`으로 관리되며, 현재 회차와 재조치 필요 문서가 분리되어 보입니다.
- `수정 업로드`는 새 문서를 만드는 방식이 아니라 기존 반려 회차에 다시 연결되어 이력이 이어집니다.
- `근로자의견청취`, `부적합사항`은 파일 업로드가 기본이고, `수동 row 추가 (보조)`는 접힘 영역으로 이동했습니다.
- HQ 승인/반려 화면에서는 회차 이력과 현재 제출본을 함께 확인할 수 있습니다.
- 문서별 `코멘트`가 추가되어 SITE와 HQ가 승인/반려와 별개로 메모를 누적할 수 있습니다.
- 날씨 카드는 실시간 조회 대신 `KST 05:00 / 12:00 스냅샷` 기준으로 표시되며, 실패 시에도 안전 문구로 종료됩니다.

## 대시보드

- SITE 대시보드는 오늘 할 일과 현재 상태를 먼저 보여줍니다.
- HQ 대시보드는 전 현장 상태, 문서 진행 현황, 날씨 요약을 모니터링 중심으로 보여줍니다.
- 날씨/미세먼지는 실시간 스트리밍이 아니라 `KST 1일 2회 스냅샷(05:00, 12:00)` 구조입니다.
- 카드에는 `스냅샷 기준 시각(KST)`과 `외부 조회 갱신 시각`이 함께 표시됩니다.
- 외부 연동 실패 시에도 화면은 멈추지 않고 `정보 없음`, `위치 미설정`, `일시적 조회 실패` 중 하나로 안전하게 표시됩니다.

## 공지사항

- 공지 등록, 조회, 댓글로 전달사항을 공유합니다.
- SITE 화면 상단에는 최신 공지 제목이 티커로 표시됩니다.
- 현장은 공지를 빠르게 확인하고, 본사는 공지 등록과 수정 관리에 주로 사용합니다.

## 안전보건 방침 및 목표

- HQ는 본사 방침/목표 파일을 업로드하고 최신 파일을 교체 관리합니다.
- SITE는 기본적으로 현장 또는 본사 기준 자료를 조회합니다.
- 업로드된 PDF와 이미지는 `파일 보기`가 기본 UX입니다.
- PDF는 인라인으로 열리고, 이미지는 즉시 미리보기로 열립니다.
- `다운로드`는 보조 기능으로 유지되며, 화면에서 먼저 확인한 뒤 필요한 경우 저장하면 됩니다.

## 안전 교육

- 교육자료를 목록으로 확인합니다.
- HQ 권한으로 교육자료 업로드가 가능합니다.
- 현장은 최신 교육자료 조회와 내려받기에 주로 사용합니다.

## 안전 점검

- 문서취합의 안전점검(INSPECTION) 파일을 게시판 형태로 확인합니다.
- 점검 문서별 코멘트로 간단한 소통이 가능합니다.
- 점검 자료는 문서취합과 연결된 상태를 기준으로 확인합니다.

## 내 현장 문서 / 문서 취합 현황

- SITE는 `현장 문서취합 (Requirement 기반)` 화면에서 오늘 제출할 문서와 재업로드 문서를 확인합니다.
- `문서 업로드`는 일반 제출, `문서 수정 업로드`는 반려된 회차를 다시 제출할 때 사용합니다.
- 수정 업로드는 일반 업로드와 같은 파일 형식을 사용하되, 기존 `instance`에 다시 연결되어 이력이 유지됩니다.
- `재조치 필요 문서`는 실제 반려 이력이 있는 항목만 별도로 강조됩니다.
- 반려 사유에는 내부 시스템 문구가 아니라 실제 사용자 메모만 노출되도록 정리되었습니다.
- 업로드된 PDF/이미지/TXT는 `보기`로 바로 열리고, 브라우저 미리보기가 어려운 형식만 다운로드로 전환됩니다.
- 문서 이력 모달 하단의 `현재 문서 코멘트`에서 현장과 본사가 문서별 메모를 누적할 수 있습니다.

## HQ 문서 검토 / 승인 / 반려

- HQ는 문서취합현황에서 현장별 상태를 보고 회차 상세로 진입합니다.
- 회차 상세에서는 `현재 제출본`, `검토 / 결재`, `회차 히스토리`를 한 화면에서 확인합니다.
- 승인 시 코멘트는 선택 입력, 반려 시 사유 입력이 권장됩니다.
- 승인/반려와 별도로 하단 `문서 코멘트`에서 SITE/HQ 공통 메모를 남길 수 있습니다.
- 코멘트는 삭제/수정 없이 누적만 되며, 승인/반려 흐름과 독립적으로 저장됩니다.

## 위험성평가 승인 흐름

- 위험성평가 반영은 `SITE 승인 -> HQ 최종승인` 2단계 구조입니다.
- SITE 승인 전에는 HQ 최종승인을 할 수 없습니다.
- HQ 최종승인 전에는 최종 반영 DB나 최종 문서 반영 대상으로 간주하지 않습니다.
- 동일 `hazard + countermeasure` 조합은 중복 반영되지 않도록 정리됩니다.

## 근로자의견청취

- 기본 운영 방식은 관리대장 파일(xlsx/csv) 업로드입니다.
- 업로드 후 의견 항목이 자동 파싱되며, 생년월일/휴대전화/의견종류를 함께 관리합니다.
- 직접 한 줄씩 넣어야 하는 예외 상황만 `수동 row 추가 (보조)`를 펼쳐 사용합니다.
- 기본 워크플로우는 `현장승인 -> 본사확인 -> 포상후보`입니다.
- 현장에서는 업로드와 1차 정리, 본사는 검토와 후보 판단에 주로 사용합니다.

## 부적합사항

- 기본 운영 방식은 부적합사항 관리대장(xlsx) 업로드입니다.
- 업로드 후 항목이 파싱되어 목록으로 보이고, 원본 다운로드와 PDF 보기, 개선조치 편집이 가능합니다.
- 예외적으로 직접 입력해야 할 경우만 `수동 row 추가 (보조)`를 펼쳐 사용합니다.
- 현장에서는 업로드와 개선조치 보완, 본사는 이력 확인과 관리 상태 점검에 주로 사용합니다.

## 문서 탐색

- PDF 중심으로 문서를 검색하고 바로 열람합니다.
- 기준자료와 현장 문서를 함께 탐색할 수 있습니다.
- 현장과 본사 모두 빠르게 참고 문서를 찾는 용도로 사용합니다.

## 소통자료

- SITE 현장 간 소통용 자료를 공유하고 미확인 건수를 확인합니다.
- 필요한 자료를 목록으로 보고 확인 상태를 추적합니다.

## 운영 아이디어 제안

- 운영 개선 아이디어를 등록하고 이력을 확인합니다.
- 현장/본사 누구나 개선 아이디어를 남길 수 있습니다.

## 설정

- 현장 정보와 설정 값을 확인하거나 변경합니다.
- HQ의 문서 설정관리에서는 문서 항목, 주기, 동적 메뉴를 관리합니다.
- 설명서용 화면 예시 이미지는 지정 계정(`hq01`)으로 업로드할 수 있습니다.

## 동적 메뉴(본사 설정)

- HQ 설정에서 게시판형/표형 메뉴를 추가할 수 있습니다.
- 메뉴 순서는 드래그앤드롭 후 `메뉴 순서 저장`으로 확정됩니다.
- 동적 메뉴는 `안전보건 방침 및 목표` 아래에 노출됩니다.

## 구글설문 연동 운영(A안)

- 구글폼 응답을 CSV로 내려받아 `근로자의견청취`에 업로드합니다.
- 자동 로그인 방식(아이디/비밀번호 저장)은 권장하지 않습니다.
- 운영 권장 순서:
  1) 구글폼 응답 수집
  2) CSV 다운로드
  3) BESMA 업로드 및 검토

## 스크린샷 촬영 가이드

- 사용설명서 화면은 메뉴별 스크린샷 카드를 자동으로 표시합니다.
- 실제 이미지가 없으면 페이지에 `스크린샷 준비중` 자리표시와 촬영 가이드가 자동 표시됩니다.
- 실제 운영 화면 캡처는 `frontend/public/guide-shots/README.md` 파일명 규칙에 맞춰 배치하면 됩니다.
- 이번 문서 기준으로 우선 필요한 대표 캡처는 `문서취합`, `근로자의견청취`, `부적합사항` 3장입니다.
